Requisitos de Instalación

  • Composer
  • PHP 7.2.4

Instalar Laravel

Ejecutar el siguiente comando en la Terminal

composer global require "laravel/installer"

Iniciar Proyecto

Crear el primer proyecto e iniciar el servidor de prueba

Crear Proyeto

Para crear un nuevo proyecto en Laravel, se debe estar ubicado en la ruta donde se desea que este el proyecto y ejecutar el siguiente comando:

laravel new NombreProyecto

Iniciar servidor de prueba

Si tiene PHP instalado localmente y desea utilizar el servidor de desarrollo incorporado de PHP para servir su aplicación, puede usar el comando serve de Artisan. Este comando iniciará un servidor de desarrollo en http: // localhost: 8000:

php artisan serve

Para ejecutar este comando se debe estar ubicado en la carpeta del proyecto

Configuracion Inicial

Nombre de la aplicación

Para cambiar el nombre de la aplicación se ejecuta el siguiente comando en la terminal

php artisan app:name NombreAplicacionn

Zona Horaria

Para configurar la zona horaria o timezone vamos al archivo config/app.php y buscamos la siguiente línea y la cambiamos por nuestra zona.

'timezone' => 'UTC',

podemos buscar nuestra zona horaria en http://php.net/manual/es/timezones.php

Luego de encontrarla quedaría asi:

'timezone' => 'America/Bogota',

Lenguaje

Para el lenguaje por defecto de nuestra aplicación vamos al archivo config/app.php y buscamos la siguiente línea

'locale' => 'en' y luego la cambiamos por 'locale' => 'es',

Añadir nuestro lenguaje

Luego de cambiar el lenguaje en config/app.php debemos añadir la carpeta con los archivos en el lenguaje seleccionado en la ruta resources/Lang

En la URL https://github.com/caouecs/Laravel-lang se encuentran algunos lenguajes que podemos usar para nuestro proyecto, podemos buscar el archivo es en la ruta src

Laravel Mix

Configurar Laravel Mix para Compilar archivos SASS y Javascript y para recargar el navegador automáticamente

Requisitos

  • Node JS

Instalar dependencias

Desde la Terminal y ubicados en la carpeta del proyecto, ejecutar el comando npm install, con el fin de instalar todas las dependencias del archivo package.json entre las que se incluyen

  • laravel-mix
  • vue
  • webpack-dev-server
  • browser-sync
  • browser-sync-webpack-plugin

Luego de ejecutar npm install se crea la carpeta node_modules la cual es ignorada al subir al repositorio git. Esto se configura en el archivo .gitignore

Compilar

En el archivo webpack.mix.js se encuentra la configuración de Laravel Mix.

En el siguiente archivo se observa la configuración para compilar el código existente en el archivo app.js y el archivo app.scss .Se define la ruta de origen y la ruta de destino.

mix.js('resources/assets/js/app.js', 'public/js')
.sass('resources/assets/sass/app.scss', 'public/css');

Compilación Manual

Ejecutar el comando

npm run dev

Compilación Automática

La compilación automática consiste en que la terminal escucha de manera activa cualquier cambio que se realice en los archivos app.js y app.scss. Al detectar un cambio se compila automáticamente. Para ello en la terminal se ejecuta el comando

npm run watch

Configurar BroserSync

En el archivo webpack.mix.js escribir el siguiente código

						

mix.browserSync({
	proxy: 'localhost:8000',
	browser: 'chrome.exe'
}

En Proxy se escribe la dirección URL donde se ejecuta el proyecto

En Browser se escribe el nombre del navegador con el que se desea trabajar

SQL Server

Conexión de Laravel 5.6 con SQL Server, teniendo PHP 7.2.4

Requisitos Windows

Microsoft Drivers 5.2 for PHP for SQL Server

Microsoft ODBC Driver 17 for SQL Server

Compatibilidad de los Drivers

Microsoft Drivers 5.2 for PHP for SQL Server

Este Driver es compatible con las siguientes versiones de PHP

  • 7.0.0+
  • 7.1.0+
  • 7.2.2+
  • 7.2.1+

Este Driver es compatible con las siguientes versiones de SQL Server

  • SQL Server 2008 R2
  • SQL Server 2012
  • SQL Server 2014
  • SQL Server 2016
  • SQL Server 2017

Este driver es compatible con los siguientes Sistemas Operativos

  • Windows 8.1
  • WIndows 10
  • Windows Server 2012
  • WIndows Server 2012 R2
  • Windows Server 2016

Microsoft ODBC Driver 17 fot SQL Server

Este Driver es compatible son las siguientes versiones del Driver de PHP para SQL Server

  • Microsoft Drivers 5.2 for PHP for SQL Server

Instalación

Instalar Microsoft Drivers 5.2 for PHP for SQL Server y Microsoft ODBC Driver 17 for SQL Server

Instalar Microsoft Drivers 5.2 for PHP for SQL Server

Descargar Drivers desde el siguiente Link Microsoft Drivers 5.2 for PHP for SQL Server​

Una vez descargado, se ejecuta el archivo SQLSRV52.EXE y se descomprimen las librerías en alguna carpeta seleccionada.

Selección de archivos

Para saber que archivos debemos utilizar debemos tener en cuenta lo siguiente

  • Architecture
  • Thread Safety

Lo anterior lo podemos verificar si ejecutamos phpinfo()

Architecture

Thread Safety

Para este caso necesitaremos los siguientes archivos

  • php_sqlsrv_72_ts_x86.dll
  • php_pdo_sqlsrv_72_ts_x86.dll

Que deben ser copiados en la siguiente ruta

Para XAMPP en C:\xampp\php\ext

Para WAMP en C:\wamp\bin\php\php7.2.4\ext

Luego debemos incluir la referencia a estos drivers en el archivo php.ini en la sección de extensiones

						
extension=php_pdo_sqlsrv_72_ts_x86.dll
extension=php_sqlsrv_72_ts_x86.dll
